# Docker installation guide (Linux, Windows, macOS)

This guide covers installing **Docker** and a **graphical UI** where Docker provides one. Commands and package names change over time; use the official links for the exact steps for your OS version.

---

## What to install

| Platform | Typical choice | Built-in GUI |
| -------- | -------------- | ------------ |
| **Windows** | [Docker Desktop](https://docs.docker.com/desktop/setup/install/windows-install/) | Yes (Docker Desktop) |
| **macOS** | [Docker Desktop](https://docs.docker.com/desktop/setup/install/mac-install/) | Yes (Docker Desktop) |
| **Linux** | [Docker Engine](https://docs.docker.com/engine/install/) **or** [Docker Desktop for Linux](https://docs.docker.com/desktop/setup/install/linux/) | Desktop app only with **Docker Desktop**; Engine alone is CLI-only |

- **Docker Desktop** bundles the daemon, CLI, Compose, and a **desktop GUI** (dashboard, containers, images, settings).
- **Docker Engine** (Linux) is the open-source daemon + CLI; there is **no official “Docker GUI”** in that package—people use the CLI, a browser UI like [Portainer](https://docs.portainer.io/start/install), or install **Docker Desktop for Linux** if they want the same app as on Windows/Mac.

---

## Windows

### 1. System requirements

- Windows 10 64-bit (build 19041+) or Windows 11.
- [WSL 2](https://learn.microsoft.com/en-us/windows/wsl/install) is required for the recommended Docker Desktop backend.

### 2. Install Docker Desktop (includes GUI)

1. Read: [Install Docker Desktop on Windows](https://docs.docker.com/desktop/setup/install/windows-install/).
2. Download the installer from [Docker Desktop for Windows](https://docs.docker.com/desktop/setup/install/windows-install/#install-docker-desktop-on-windows).
3. Run the installer, enable WSL 2 if prompted, and restart if asked.
4. Start **Docker Desktop** from the Start menu. When the whale icon shows the engine is running, the GUI is available from the system tray or the Docker Desktop window.

### 3. Verify (PowerShell or Command Prompt)

```powershell
docker version
docker compose version
```

## macOS

### 1. Choose the right build

- **Apple Silicon (M1/M2/M3/…):** use the **Apple Chip** installer.
- **Intel Macs:** use the **Intel Chip** installer.

### 2. Install Docker Desktop (includes GUI)

1. Read: [Install Docker Desktop on Mac](https://docs.docker.com/desktop/setup/install/mac-install/).
2. Download `.dmg` from [Docker Desktop for Mac](https://docs.docker.com/desktop/setup/install/mac-install/#install-docker-desktop-on-mac).
3. Drag **Docker** to **Applications**, open it, and complete the first-run setup.

### 3. Verify (Terminal)

```bash
docker version
docker compose version
```

## Linux

You can use **only the Engine** (CLI, common on servers) or **Docker Desktop** (same GUI experience as Windows/Mac, where supported).

### Option A — Docker Engine + CLI (no official Docker GUI)

1. Pick your distribution: [Install Docker Engine](https://docs.docker.com/engine/install/).
2. Follow the distro page (Ubuntu, Debian, Fedora, etc.) for repository setup, install `docker-ce`, CLI, and plugins.
3. [Post-install steps](https://docs.docker.com/engine/install/linux-postinstall/) (optional): add your user to the `docker` group so you can run `docker` without `sudo`:

   ```bash
   sudo usermod -aG docker $USER
   ```

   Then sign out and back in (or reboot).

4. **Graphical tools (optional):** run [Portainer](https://docs.portainer.io/start/install/server/docker/linux) as a container, or use your editor/IDE integrations—not part of Engine itself.

### Option B — Docker Desktop for Linux (includes GUI)

1. Read: [Install Docker Desktop on Linux](https://docs.docker.com/desktop/setup/install/linux/).
2. Check **supported platforms** (Ubuntu, Debian, Fedora, and specific versions are listed in Docker’s docs).
3. Install the `.deb` or `.rpm` (or AppImage, depending on current documentation) and start **Docker Desktop** from your app menu.

### Verify (terminal)

```bash
docker version
docker compose version
```

## After installation (all platforms)

- **Sign in:** Docker Desktop may prompt for a Docker Hub account; signing in is optional for local development but useful for pulling private images.
- **Resources:** In Docker Desktop, you can tune CPUs, memory, and disk in **Settings** → **Resources**.
- **Compose:** This project uses [docker compose](../docker-compose.yaml); `docker compose` is included with Docker Desktop and with a typical modern Engine install.

## Quick reference

| Task | Command |
| ---- | ------- |
| Test the daemon | `docker run hello-world` |
| See running containers | `docker ps` |
| Open Compose app | `docker compose up` (from the folder with `docker-compose.yaml`) |
